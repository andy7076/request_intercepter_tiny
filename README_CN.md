# Request Interceptor Tiny

**[English](./README.md) | 中文**

**轻量级的接口模拟工具**

Request Interceptor Tiny 是一款专为开发者设计的 Chrome 扩展，旨在提供简单、高效的接口数据模拟（Mock）功能。它可以拦截浏览器中的 Fetch 和 XHR 请求，并将其响应替换为自定义内容，帮助前端开发者在后端接口未就绪或需要特定测试场景时进行开发和调试。

## ✨ 主要特性

- **🔌 接口模拟 (Mock Response)**  
  拦截浏览器中的 Fetch/XHR 请求，并返回您自定义的响应数据。支持通配符 `*` 匹配 URL。

  > **注意：** 浏览器 Network 面板仍会显示原始请求/响应（透明拦截），但您的应用会接收到 Mock 数据。

  > **范围说明：** 仅支持 Fetch/XHR 请求。地址栏直接打开页面、链接跳转、iframe `src` 等页面导航请求不支持。

- **🛠️ 强大的响应内容编辑器**  
  内置专业级编辑器（基于 CodeMirror），支持：
  - **内容类型更灵活**：支持文本、JSON、HTML、XML 等多种响应内容。
  - **智能辅助**：实时 JSON 语法校验、语法高亮、代码折叠/展开、行号显示。
  - **增强型搜索替换**：全新的搜索体验，支持正则匹配、即时高亮、智能光标跟随，修改内容时互不干扰。
  - **沉浸式编辑**：支持弹窗内全屏编辑及独立标签页（New Tab）模式，自适应布局，轻松应对大型响应内容。

- **🖥️ 侧边栏模式 (Side Panel)**
  利用 Chrome Side Panel API，提供不打扰浏览体验的持久化操作界面，无需反复打开弹窗。

- **🌍 多语言支持 (Internationalization)**
  原生支持中文和英文界面，自动跟随浏览器语言设置，无缝切换。

- **📊 请求日志监控**  
  实时记录命中的 Fetch/XHR 请求详情。支持查看响应体内容以及对比原始数据与 Mock 数据（Diff 视图），让您清楚地掌握 Mock 效果。

- **🔁 规则管理**
  - 支持规则的快速创建、编辑、删除和一键启用/禁用。
  - **高级设置**：支持为每条规则单独配置请求方法、匹配方式、优先级、响应状态码、响应延迟和响应头。
  - **拦截开关**：支持全局开启或关闭拦截。关闭后，不再向页面注入拦截脚本。
  - **导入/导出**：支持将规则导出为 JSON 文件分享给团队成员，或从文件导入现有规则。

- **⚡ cURL 快速导入**  
  从浏览器 DevTools 复制请求的 cURL 命令，一键解析并创建规则。自动发起请求获取真实响应作为 Mock 数据模板。

- **🎨 极简现代化 UI**  
  支持**跟随系统**、**浅色**、**深色**三种主题模式，切换丝滑流畅。融合玻璃拟态风格，界面精致、交互流畅。

- **🚀 轻量安全**  
  基于 Chrome Manifest V3 规范开发，纯原生 JavaScript 实现，无需额外依赖，性能零损耗。

## 📥 安装指南

> **推荐：** [从 Chrome 应用商店安装](https://chromewebstore.google.com/detail/request-interceptor-tiny/nhofohmjmciklmcompcjoemkbahdipco)

### 手动安装

1. **获取源码**  
   克隆或下载本项目到本地。

2. **加载扩展**
   - 在 Chrome 浏览器地址栏输入 `chrome://extensions/` 并回车。
   - 打开右上角的 **"开发者模式" (Developer mode)** 开关。
   - 点击左上角的 **"加载已解压的扩展程序" (Load unpacked)** 按钮。
   - 选择本项目的根目录。

3. **开始使用**  
   安装完成后，点击浏览器工具栏的扩展图标即可使用。

## 📖 使用说明

1. **添加拦截规则**
   - 点击扩展图标打开界面。
   - 切换到 **"添加规则"** 标签页。
   - **方式一：手动填写**
     - **规则名称**：给规则起一个易于识别的名字。
     - **URL 匹配模式**：输入要拦截的请求地址，支持 `*` 通配符（例如：`*://api.example.com/v1/users*`）。
     - **响应内容**：输入您期望返回的响应内容，支持文本、JSON、HTML、XML 等。
     - **高级设置**：
       当您需要更精确地匹配请求，或者希望模拟更真实的接口响应时，可以展开这一部分进行配置。
     - **请求方法**：
       可将规则限制为 `GET`、`POST`、`DELETE` 等特定方法；如果同一份 Mock 需要匹配所有方法，保持 `All` 即可。
     - **匹配方式**：
       `Contains` 适合大多数快速匹配场景，`Wildcard` 适合配合 `*` 通配符使用，`Exact` 则要求请求 URL 完全一致才会命中。
     - **优先级**：
       数值越高越先命中。当多条规则可能同时匹配同一个请求时，可以通过优先级明确哪条规则优先生效。
     - **响应状态码**：
       可设置返回的 HTTP 状态码，例如 `200`、`404`、`500`，便于模拟成功、业务异常或服务端错误等情况。
     - **响应延迟**：
       可为返回结果增加毫秒级延迟，适合测试加载中、弱网、超时处理等场景。
     - **响应头**：
       支持按行添加响应头，例如 `Content-Type`、`Cache-Control`。当您希望浏览器或业务代码把 Mock 内容按 HTML、XML、JSON、纯文本、下载文件或特定缓存策略处理时，这里会很有用。
   - **方式二：从 cURL 导入**
     - 在浏览器 DevTools 的 Network 面板中，右键点击请求 → "Copy as cURL"。
     - 点击 **"从 cURL 导入"** 按钮，粘贴命令后点击解析。
     - 扩展会自动发起请求获取真实响应，并填充规则名称、URL 模式和响应内容。
   - 点击 **"保存规则"**。

2. **验证效果**
   - 打开或刷新目标网页。
   - 当网页发起匹配的 Fetch/XHR 请求时，扩展会自动拦截并返回您设置的数据。
   - 您可以在扩展的 **"请求日志"** 面板中查看拦截记录。

3. **管理规则**
   - 在 **"规则列表"** 面板中，您可以随时开启/关闭特定规则，或进行编辑和删除操作。
   - 点击右上角的 **导入/导出** 按钮可以进行规则的备份与迁移。
   - 在 **设置** 中，您可以通过 **拦截开关** 全局开启或关闭拦截。切换后，如需让已打开页面完全生效，请刷新对应页面。
